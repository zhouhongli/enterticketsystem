## 1. 制作目标

本制作稿用于把 `09` 号详细页级脚本转化为可直接制作 PPT 的页面规格。与页级脚本相比，本稿更强调：

1. 每页最终放到 PPT 上的文字。
2. 哪些内容进入讲师备注，而不是放到页面正文。
3. 每页采用什么版式和图形。
4. 每页引用哪个项目文档作为素材来源。
5. 如何控制信息密度，避免页面变成文档截图。

成品定位为“案例型培训课件”，不是产品汇报、工具教程或代码教学。

## 2. 母版与通用规范

### 2.1 页面规格

| 项目 | 规范 |
| --- | --- |
| 页面比例 | `16:9` |
| 背景 | 白色 `#FFFFFF`，必要时使用极浅灰分区 `#F6F8FA` |
| 主色 | 科技蓝 `#1464A5` |
| 强调色 | 绿色 `#20A36A` 表示确认、通过、可执行 |
| 风险色 | 橙色 `#F28C28` 表示风险、边界、发现项 |
| 正文色 | 深灰 `#333333` |
| 标题色 | 近黑 `#111111` |
| 字体 | 微软雅黑 |
| 页脚 | 左侧放章节短名，右侧放页码；素材页可在页脚中放来源文档简称 |

### 2.2 版式类型

| 版式编号 | 名称 | 用途 |
| --- | --- | --- |
| `L1` | 封面/观点页 | 封面、章节观点、总结页 |
| `L2` | 双栏对照 | 讲什么/不讲什么、聊天确认/文档落盘、风险路径/推荐路径 |
| `L3` | 时间线/流程线 | 项目主线、状态流、M0-M6 计划 |
| `L4` | 三列角色/能力卡 | 客户、客服、管理员；技术/数据/API |
| `L5` | 文档片段页 | PRD、SRS、API、JSON、测试片段 |
| `L6` | 矩阵/表格页 | 权限矩阵、追踪矩阵、测试用例、E2E |
| `L7` | 中心辐射图 | 观察点、Agent 作用、规则追踪 |
| `L8` | 练习页 | 学员练习和讨论 |

### 2.3 可见文本控制

- 普通页面主标题不超过 `18` 个汉字。
- 每页主观点最多 `1` 句。
- 列表项建议 `4-7` 条。
- 超过 `8` 条的内容应改为两栏、分组或缩略矩阵。
- 代码、JSON、API 示例只保留最能说明问题的片段。
- 讲师要讲的背景、解释、转场放入备注，不强行放到页面。

### 2.4 图形风格

- 图标使用线性风格，避免卡通化。
- 文档片段用浅灰底、细边框、等宽局部代码块。
- 流程线用蓝色主线；确认节点用绿色；风险/排除项用橙色。
- 不使用复杂背景、渐变大面积装饰或拟人机器人图。
