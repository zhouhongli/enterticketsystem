## 0. 历史版本说明

本文记录第一轮 `22` 页母版初稿的页面设计依据。经用户审阅，原结构对三种模式的完整流程、拟真产物、质量责任、`Spec-Driven Development` 实践与 Agent 纠偏过程呈现不足，导致演示文稿信息量和讲授深度不够。后续制作应以新版页级脚本与独立讲师稿为依据，本文不继续增补为成品基线。

## 1. 制作目标

本脚本将已确认培训大纲转化为可直接制作演示文稿的页面设计输入。它以完整课程母版为目标，不预先绑定讲授时长，不设置现场互动或练习环节；后续可依据实际授课时间，删减扩展页面或压缩讲解深度，派生短版演示文稿。

母版需传达的核心判断为：

> AI Agent 改变的是软件开发中可委派活动的粒度与反馈密度；人的目标判断、约束设定、结果审查与责任承担仍是软件工程成立的前提。

- 全部可见文字使用中文表达；框架专名 `Superpowers`、`Spec Kit`、`OpenSpec` 与必要缩写除外。
