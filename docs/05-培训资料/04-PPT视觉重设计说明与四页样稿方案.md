## 1. 重设计原因

旧版初稿完成了课程内容转译和基础规范落实，但整体视觉表现偏向规则化讲义：页面卡片结构重复，封面与关键页缺乏视觉层次，主观点不足以形成授课节奏。新版需要在不削弱培训信息完整度的前提下，建立更成熟的商务科技演示语言，让管理类听众能够快速识别演进主线、关键变化与责任边界。

## 2. 已确认的新方向

| 维度 | 新版策略 |
| --- | --- |
| 主叙事 | 采用演进叙事：瀑布/V -> 敏捷/DevOps -> AI Agent 协作。 |
| 关系表达 | 不描述为替代，而是说明工程能力逐步叠加与协作颗粒度继续缩小。 |
| 内容密度 | 培训型信息完整；听众脱离讲解仍能理解主要结论。 |
| 页面规模 | 不再预设页数上限，先保证三种模式的流程、产物、质量责任与案例解释完整。 |
| 视觉风格 | 纯白底、蓝色主结构、橙色风险强调、绿色价值/验证强调。 |
| 主视觉选择 | 不使用照片、插画或主视觉图片，依靠高级排版、留白、线条与色块。 |
| 字号层级 | 关键页允许 `36-44pt` 核心标题；普通内容页保持标题 `28pt`、正文 `18pt`。 |
| 图形策略 | 使用贯穿时间线、泳道、阶段放大图、责任门禁图，减少重复卡片。 |

## 3. 新版内容结构方向

| 模块 | 重设计重点 |
| --- | --- |
| 导入 | 以大标题和演进路径建立“模式变化”问题，而非罗列课程信息。 |
| 模式演进 | 使用一条贯穿式时间线解释三类模式各自解决的问题与继承关系。 |
| Agent 定义 | 用任务颗粒度和反馈闭环表达 Agent 的当前可落地定位。 |
| 生命周期 | 以生命周期泳道和阶段放大页交替展示 Agent 参与方式及人工责任。 |
| 价值与风险 | 使用视觉对照页，显示速度收益与错误放大效应同时存在。 |
| 管理收束 | 以五项控制原则和管理者五问形成可记忆的最终框架。 |

## 4. 第一轮四页视觉样稿

第一轮曾以四张关键页验证新版视觉语言。样稿已完成，但用户进一步确认：问题根源在于页级脚本信息不足，后续必须先充实内容结构，再制作完整新版 PPT。

| 样稿页 | 主题 | 需要验证的设计能力 |
| --- | --- | --- |
| `S01` | 新版封面 | 大字号标题、留白、演进线索与商务科技质感。 |
| `S02` | 三模式演进总图 | 一条主线下的信息组织、三类模式关系与能力叠加。 |
| `S03` | 全生命周期 Agent 参与图 | 信息完整度、流程阅读性和人工门禁表达。 |
| `S04` | 治理原则结论页 | 强观点收束、蓝橙绿强调和可记忆总结效果。 |

## 5. 样稿验收关注点

| 关注点 | 判断方式 |
| --- | --- |
| 视觉层次 | 第一眼是否能看到主观点，而不是先看到大量边框。 |
| 信息完整 | 不听讲解时，页面是否仍能解释其核心含义。 |
| 页面节奏 | 四页是否体现封面、解释、全景与结论四种不同节奏。 |
| 科技商务感 | 无照片情况下，是否依靠排版与线条形成专业品质。 |
| 后续扩展性 | 设计语言是否适合承载内容更完整的新版母版。 |
