# 企业售后工单系统 V1.0.0 手工测试方案

> **环境**: http://localhost:8000
> **浏览器**: Chrome / Edge
> **目标**: 验证 M0-M5 全部功能按 PRD 正常工作

---

## 一、前置准备

| 步骤 | 操作 | 预期 |
|---|---|---|
| 1 | 设置环境变量并重启服务 | `INITIAL_ADMIN_USERNAME=admin`, `INITIAL_ADMIN_EMAIL=admin@example.com`, `INITIAL_ADMIN_PASSWORD=Admin@12345` |
| 2 | 启动服务（已启动） | http://localhost:8000 可访问 |
| 3 | 清理数据（可选） | 备份或删除 `backend/data/store.json`，重启后自动创建空数据文件 |

---

## 二、测试用例

### 模块 1：认证与注册（任意角色）

| # | 用例 | 操作步骤 | 预期结果 |
|---|---|---|---|
| A1 | 未登录访问主页 | 浏览器打开 http://localhost:8000 | 自动跳转至 `/pages/login.html` |
| A2 | 正常注册-客户 | 注册页输入用户名/邮箱/密码/确认密码 → 提交 | 提示注册成功，跳转登录页 |
| A3 | 注册-密码不一致 | 注册时两次密码不同 → 提交 | 红框标出不一致的密码字段 |
| A4 | 注册-空用户名 | 用户名为空 → 提交 | 提示用户名不能为空 |
| A5 | 注册-重复用户名 | 用已有用户名再次注册 | 提示用户名已存在 |
| A6 | 注册-重复邮箱 | 用已有邮箱再次注册 | 提示邮箱已被注册 |
| A7 | 正常登录 | 输入正确用户名/密码 → 登录 | 跳转对应角色的工单列表页，导航栏显示角色身份 |
| A8 | 登录-错误密码 | 输入正确用户名+错误密码 | 提示用户名或密码错误 |
| A9 | 登录-不存在的账号 | 输入不存在的用户名 | 提示用户名或密码错误 |
| A10 | 正常退出 | 点击导航栏"退出" | 跳转登录页 |
| A11 | 退出后访问受保护页面 | 退出后手动输入客户工单列表 URL | 跳转登录页 |
| A12 | 会话过期（8h TTL） | 登录后等待 8 小时再操作（可跳过） | 跳转登录页 |

### 模块 4：内部用户（客服/管理员）- 工单处理

| # | 用例 | 操作步骤 | 预期结果 |
|---|---|---|---|
| D1 | 登录客服账号 | 用 B3 创建的客服账号登录 | 进入内部工单列表页 |
| D2 | 工单列表-按状态筛选 | 点击"待分配"/"处理中"等筛选按钮 | 列表仅显示对应状态的工单 |
| D3 | 工单列表-全部状态 | 不选择筛选条件 | 显示全部工单 |
| D4 | 工单详情 | 点击工单进入详情 | 显示标题、描述、分类、状态、负责人、全部留言 |
| D5 | 内部留言 | 客服在详情页输入留言 → 提交 | 留言显示在下方，标注客服身份 |
| D6 | 状态流转: 待分配 → 处理中 | 客服/管理员点击"开始处理" | 状态变为"处理中"，记录审计日志 |

| D10 | 已关闭工单不可改状态 | 尝试推进状态 | 提示状态不可变更 |
| D11 | 分配工单（仅管理员） | 管理员点击"分配" → 选择客服 → 确认 | 工单分配给指定客服 |
| D12 | 重新分配（仅管理员） | 管理员重新分配给另一客服 | 负责人更新，记录审计日志 |
| D13 | 客服无法分配工单 | 客服账号无分配按钮 | 无分配按钮或点击后提示无权限 |

### 模块 6：权限隔离与安全

| # | 用例 | 操作步骤 | 预期结果 |
|---|---|---|---|
| F1 | 客户访问内部页面 | 客户登录后手动访问 `/pages/internal/tickets.html` | 被前端拦截并跳转客户工单列表 |
| F2 | 客户访问管理员页面 | 客户登录后手动访问 `/pages/internal/dashboard.html` | 被前端拦截并跳转 |
| F3 | 客服访问管理员页面 | 客服登录后访问数据看板 | 被拦截并跳转内部工单列表 |
| F4 | 客户调用内部 API | 用客户 Cookie 请求 `/api/v1/internal/tickets` | 返回 403 FORBIDDEN |
| F5 | 未登录调用 API | 无 Cookie 请求 `/api/v1/customer/tickets` | 返回 401 AUTHENTICATION_REQUIRED |
| F6 | XSS 防护 | 在标题/描述/留言中输入 `<script>alert(1)</script>` | 显示为转义文本，不执行脚本 |
| F7 | Cookie 属性 | 登录后检查浏览器开发者工具中的 Cookie | HttpOnly=true, SameSite=Lax, Secure=false（本地开发） |
| F8 | 错误响应格式 | 触发任意 API 错误（如 401） | 返回 `{ "error": { "code": "...", "message": "..." } }` 格式 |

## 三、测试执行顺序

1. **先执行模块 1**（认证）→ 为后续模块准备账号
2. **再执行模块 2**（管理员）→ 准备分类、客服、客户数据
3. **再执行模块 3**（客户）→ 用模块 2 创建的客户账号建单
4. **再执行模块 4**（内部）→ 用模块 2 创建的客服账号处理工单
5. **再执行模块 5**（数据看板）→ 用前面步骤产生的数据验证
6. **最后执行模块 6**（安全）→ 穿插在各角色间验证权限

## 四、预期数据流（端到端）

```
管理员创建分类 (B10)
      ↓
客户注册 → 登录 (A2, A7)
      ↓
客户新建工单，状态=待分配 (C2)
      ↓
管理员分配工单给客服 (D11)
      ↓
客服点击"开始处理"，状态=处理中 (D6)
      ↓
客服公开留言 (D5)
      ↓
客户在详情页查看留言 (C9)
      ↓
客服点击"标记已解决"，状态=已解决 (D7)
      ↓
客服/管理员点击"关闭工单"，状态=已关闭 (D8)
      ↓
管理员查看数据看板，确认统计正确 (E1-E9)
```
