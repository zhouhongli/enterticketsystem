## 1. 设计目的

本文档用于在进入代码开发前，以低保真线框方式帮助确认页面结构、信息优先级、操作入口和关键状态表现。本文档不定义最终视觉风格、颜色、字体、图标、响应式细节或前端实现方式。

低保真设计遵循以下边界：

- 不扩大 MVP 功能范围，不增加附件、通知、统计、搜索、分页、批量操作等能力。
- 页面只呈现当前角色可理解或可执行的主要操作。
- 后端权限校验仍是最终安全边界，页面隐藏或禁用按钮仅用于改善交互。
- 已确认采用 `A 线性流程型`：保持多页面、单页面单主任务、顶部页头与角色导航、列表进入详情、详情集中展示状态与可执行操作的结构。
- 若本文档与已确认的 PRD、SRS、页面与交互说明、技术设计或 API 设计发生冲突，以已确认正式基线文档为准。

## 2. 全局页面框架

登录后的客户门户和内部后台共用基础框架，但导航项按角色变化。

```text
┌──────────────────────────────────────────────────────────────┐
│ 企业售后工单系统                         用户名｜角色｜退出 │
├──────────────────────────────────────────────────────────────┤
│ 导航：当前角色可访问模块                                     │
├──────────────────────────────────────────────────────────────┤
│ 页面标题                                                     │
│ 操作反馈区：成功 / 错误 / 提示                                │
│                                                              │
│ 当前页面主体内容                                             │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 2.1 客户导航

```text
我的工单 | 新建工单 | 退出
```

### 2.2 客服导航

```text
工单列表 | 退出
```

### 2.3 管理员导航

```text
工单列表 | 分类管理 | 客服账号 | 客户账号 | 退出
```

## 3. 公共页面

### 3.1 登录页

目标：让客户、客服、管理员使用统一入口登录；未注册客户可进入注册页。

```text
┌──────────────────────────────┐
│ 企业售后工单系统             │
│ 登录                         │
│                              │
│ [ 用户名或电子邮箱       ]   │
│ [ 密码                  ]   │
│                              │
│ [ 登录 ]                     │
│                              │
│ 还没有客户账号？注册客户账号 │
│                              │
│ 提示/错误信息显示区          │
└──────────────────────────────┘
```

交互要点：

| 场景 | 页面表现 |
| --- | --- |
| 登录成功 | 根据角色进入“我的工单”或“工单列表”。 |
| 登录失败 | 在表单下方显示统一失败提示。 |
| 点击注册 | 跳转客户注册页。 |
| 已登录访问登录页 | 可直接跳转当前角色首页。 |

### 3.2 客户注册页

目标：只创建客户账号，不暴露客服或管理员注册入口。

```text
┌──────────────────────────────┐
│ 企业售后工单系统             │
│ 注册客户账号                 │
│                              │
│ [ 用户名                 ]   │
│ [ 电子邮箱               ]   │
│ [ 密码                  ]   │
│ [ 确认密码              ]   │
│                              │

│                              │
│ 字段错误 / 注册结果提示       │
└──────────────────────────────┘
```

交互要点：

| 场景 | 页面表现 |
| --- | --- |
| 注册成功 | 跳回登录页，并提示“注册成功，请登录”。 |
| 字段校验失败 | 在对应字段附近展示错误，保留用户名和邮箱，清空密码。 |
| 返回登录 | 不保存未提交内容。 |

## 7. 关键流程低保真流转

### 7.1 客户提交工单

```text
登录页
  -> 客户注册页
  -> 注册成功后返回登录页
  -> 我的工单页
  -> 新建工单页
  -> 创建成功后进入客户工单详情页
```

### 7.2 管理员准备运行基础

```text
登录页
  -> 工单列表页
  -> 分类管理页：新增有效分类
  -> 客服账号页：创建客服
  -> 工单列表页：等待客户新工单
```

### 7.3 工单分配与处理闭环

```text
内部工单列表页
  -> 内部工单详情页
  -> 管理员分配客服
  -> 负责客服进入详情页
  -> 留言 / 开始处理
  -> 标记已解决
  -> 确认关闭
  -> 已关闭只读
```

## 8. 已确认的线框评审结论

以下问题不改变 MVP 功能范围，已随 `A 线性流程型` 的选择一并确认为前端实现时的默认取舍：

| 编号 | 问题 | 确认结论 |
| --- | --- | --- |
| WF-Q01 | 内部工单详情页中“分配区、状态操作区、留言区、操作记录区”的上下顺序是否符合使用习惯？ | 保持本稿顺序：先概要，再分配/状态，再沟通，再记录。 |
| WF-Q02 | 非负责客服无权处理时，是隐藏按钮还是显示禁用按钮并说明原因？ | 显示只读提示，关键操作按钮隐藏，减少误操作；后端仍负责最终权限校验。 |
| WF-Q03 | 客户详情页是否需要展示“更新时间”？ | 展示更新时间，便于客户知道工单最近是否有变化；仍不展示负责人信息。 |
| WF-Q04 | 管理员管理页的新增表单是否放在列表上方？ | 新增表单放在列表上方，便于 MVP 首次配置和小规模维护。 |
| WF-Q05 | 关闭工单确认是否使用浏览器原生确认框还是自定义弹窗？ | MVP 可使用浏览器原生确认框；后续视觉优化时再替换。 |

## 9. 页面结构方向确认

OpenDesign 线框评审中曾并排比较以下结构方向：

| 方案 | 结构特征 | 处理结论 |
| --- | --- | --- |
| `A 线性流程型` | 多页面结构直观；每页一个主要任务；顶部页头、角色导航、内容区和反馈区清晰；适合 MVP 快速实现与验收走查。 | 已选择为本项目 MVP 页面结构方向。 |
| `B 双栏工作台型` | 左侧固定上下文与导航，右侧承载当前任务；适合页面增长后的后台体验。 | 暂不作为 MVP 默认结构，可作为后续视觉优化参考。 |
| `C 状态时间线型` | 强调工单状态流和处理历史；适合突出闭环与追溯。 | 可吸收“状态清晰可见”的表达，但不改变 A 方案整体结构。 |
| `D 收件箱处理型` | 强调内部任务队列和详情预览；适合客服高频切换处理场景。 | 暂不作为 MVP 默认结构，避免引入接近复杂工作台的交互。 |

最终确认采用 `A 线性流程型`，后续前端实现应优先遵循本文第 2 至第 7 章的页面结构、流程顺序和交互要点。其他方案仅作为局部表达参考，不应引入新的页面、入口、数据字段或超出 MVP 的交互能力。
