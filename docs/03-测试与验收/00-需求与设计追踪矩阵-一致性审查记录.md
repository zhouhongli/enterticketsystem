## 1. 审查目的

## 3. 审查方法与范围

本次审查以 SRS 中带编号的强制需求为追踪起点，逐组映射至页面交互、技术设计、数据模型和 API 设计，并补充检查角色权限、状态流转、敏感字段隔离、存储演进能力与 MVP 范围边界。

SRS 共包含 `88` 项编号需求：`AUTH` 20 项、`CAT` 6 项、`TKT` 11 项、`ASG` 7 项、`MSG` 8 项、`STS` 8 项、`LOG` 5 项、`SEC` 9 项、`DAT` 7 项、`NFR` 7 项。

## 6. 跨文档一致性核验

### 6.1 角色权限核验

| 角色 | 关键需求 | 设计核验结果 |
| --- | --- | --- |
| 客户 | 注册登录、创建工单、查看本人工单、公开留言 | API 按会话绑定客户本人资源；数据响应不包含内部记录；整体一致。 |
| 客服 | 查看全部工单、仅处理分配给自己的工单 | API 对留言与状态更新校验当前负责人；未分配或他人负责的工单只读；一致。 |
| 管理员 | 管理分类与账号、分配工单、处理任意工单 | 后端权限及审计记录设计完整；一致。 |

### 6.2 状态与操作规则核验

| 需求规则 | 数据模型 | API/技术设计 | 结论 |
| --- | --- | --- | --- |
| 新工单初始为待分配 | `unassigned` | 建单响应及创建流程约束为 `unassigned` | 一致 |
| 状态只允许线性推进 | 枚举转换表限制下一状态 | 状态更新冲突返回 `409` | 一致 |
| 已关闭工单不能继续处理 | 禁止更换负责人、状态或新增留言 | 留言、状态、分配接口均有冲突处理 | 一致 |

### 6.3 敏感信息与数据暴露核验

| 信息 | 客户响应 | 内部响应 | 存储/认证规则 | 结论 |
| --- | --- | --- | --- | --- |
| 密码与会话凭证 | 不返回 | 不返回 | 仅存密码哈希与 `token_hash` | 一致 |
| 负责人信息 | 不返回 | 返回负责人标识/用户名 | API 与页面均已明确隔离 | 一致；`F-01` 已关闭 |
| 客户邮箱 | 不涉及内部展示 | 不返回 | 用于账号身份，不作为处理详情暴露字段 | 一致 |
| 内部操作记录 | 不返回 | 管理员/内部详情可见 | AuditLog 单独保存 | 一致 |

### 6.4 MVP 范围核验

| 排除项 | 需求基线状态 | 设计核验结果 |
| --- | --- | --- |
| 附件与外部通知 | MVP 不包含 | 无必选设计或接口 |
| SLA 与统计报表 | MVP 不包含 | 无必选设计或接口 |
| 高级搜索、批量处理或导出 | MVP 不包含 | 无必选设计或接口 |
| 客户确认关闭、重开或撤销 | MVP 不包含 | 状态流无相关动作 |
| 真实关系型数据库迁移 | 后续迭代 | 当前仅定义可替换仓储边界 |

## 8. 对测试与验收方案的输入

后续测试方案至少应覆盖以下高风险规则：

| 验收关注点 | 需要验证的结果 |
| --- | --- |
| 会话与账号禁用 | Cookie 属性、8 小时过期、退出撤销、客户禁用后既有会话不可继续访问。 |
| 资源与操作授权 | 客户不能访问他人工单；客服不能处理未分配或分配给他人的工单；管理员权限正确。 |
| 状态与关闭限制 | 状态不能跳级或回退；关闭后不能再留言、分配或变更状态。 |
| 敏感字段隔离 | 客户响应不出现负责人和内部操作记录；内部响应不出现客户邮箱；所有响应不出现密码哈希和会话摘要。 |
| 持久化一致性 | 关键业务操作和审计记录原子写入；MVP 仅以单应用进程方式运行。 |
| 历史可追溯性 | 分类名称、留言发送人信息和关键变更审计能够在后续数据变化后正确保留历史显示。 |
