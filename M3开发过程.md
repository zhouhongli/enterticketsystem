## 2. M3-01 多智能体执行记录

### 2.1 Explorer 只读结论

Explorer 已完成只读分析，未修改文件。

关键结论：

- `M3-01` 的核心定位是补齐统一认证入口，让客户、客服、管理员都能通过同一登录页进入系统，并让后续受保护页面通过 `/auth/me` 获取当前身份。
- 注册接口只创建 `customer`，状态为 `active`；注册成功不自动登录、不设置 Cookie。
- 登录失败必须统一返回 `401 LOGIN_FAILED`，不能区分账号不存在、密码错误或账号不可用。
- 当前身份响应只能返回 `id`、`username`、`email`、`role`、`status`，不得返回 `password_hash`、`token_hash`、Cookie 原始值或会话字段。
- 登录成功由后端设置 `ticket_session`，前端不得保存 token。
- 退出、过期、撤销或禁用后的旧会话不得继续访问受保护资源。
- 登录页按角色跳转：客户进入客户门户，客服和管理员进入内部工单列表；管理员显示管理导航，客服不显示管理入口。

### 2.2 Worker 前端切片

Worker 已完成前端登录/注册页面切片，未修改后端、测试、管理页面逻辑或本文档。

修改文件：

实现点：

- 注册成功后跳回登录页并显示“注册成功，请登录。”。
- 注册失败或前端校验失败时保留用户名、邮箱，清空密码与确认密码。

### 2.3 主智能体后端实现

主智能体完成后端认证接口和当前身份依赖集成。

新增文件：

- `backend/app/schemas/auth.py`

修改文件：

- `backend/app/api/routes/auth.py`
- `backend/app/api/dependencies.py`
- `backend/app/api/errors.py`

实现内容：

| 接口 | 完成情况 |
| --- | --- |
| `POST /api/v1/auth/register` | 创建启用状态客户账号；使用 Argon2id 哈希；响应不含敏感字段；不设置 Cookie。 |
| `POST /api/v1/auth/login` | 支持用户名或邮箱登录；邮箱匹配大小写不敏感；登录成功创建服务端会话并设置 Cookie。 |
| `POST /api/v1/auth/logout` | 需要有效登录；撤销当前会话并清除 Cookie。 |
| `GET /api/v1/auth/me` | 通过服务端会话返回当前身份白名单字段。 |

安全处理：

- 账号不存在、密码错误、账号禁用统一返回 `LOGIN_FAILED`。
- 密码只保存 Argon2id 哈希。
- 登录响应不返回 `password_hash`、`token_hash` 或会话字段。
- Cookie 设置包含 `HttpOnly`、`SameSite=Lax`、`Path=/`、`Max-Age=28800`；本地环境 `Secure=false`。
- 会话无效、已撤销、过期或用户已禁用时返回 `AUTHENTICATION_REQUIRED`，并在存在旧 Cookie 时清除 Cookie。

### 5.5 Explorer 只读结论

Explorer 已完成只读分析，未修改文件。

关键结论：

- `M3-02` 只实现 `POST /api/v1/customer/tickets`。
- 成功建单必须同时写入 `tickets` 与 `audit_logs`，且同一 JSON 原子事务提交。
- 请求体只接受 `category_id`、`title`、`description`。
- 当前客户身份只能来自服务端会话，不接受请求体中的 `customer_user_id`、角色、操作者、状态或负责人。
- 分类必须存在且 `status=active`，否则返回 `409 CONFLICT`。
- 建单时必须保存 `category_name_snapshot`，响应中的 `category_name` 来自快照。
- 初始状态固定为 `unassigned`，负责人固定为 `null`。
- 客户建单响应不得返回负责人、客户邮箱、内部操作记录或任何认证敏感字段。
- 不做列表、详情、留言、页面动态接线或内部处理能力。
