## 2. M5-01 实现内容

### 2.1 后端错误响应契约

后端已有统一错误响应入口位于 `backend/app/api/errors.py`，响应结构为：

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "错误说明",
    "field_errors": {}
  }
}
```

`field_errors` 仅在字段校验错误等需要字段明细时返回。

本阶段新增契约测试 `backend/tests/api/test_error_response_contract.py`，覆盖：

- `422 VALIDATION_ERROR`，包含 `field_errors`。
- `401 LOGIN_FAILED`。
- `401 AUTHENTICATION_REQUIRED`。
- `403 FORBIDDEN`。
- `404 RESOURCE_NOT_FOUND`。
- `409 CONFLICT`。

### 2.2 前端请求与反馈模块

调整 `frontend/assets/js/api-client.js`：

- 将非 `2xx` API 响应统一转换为带 `status`、`code`、`fieldErrors` 的错误对象。
- 对 `VALIDATION_ERROR` 生成字段级中文摘要。
- 对登录失败、未认证、无权限、资源不存在、状态冲突和存储错误提供统一中文提示。
- 对网络连接失败给出本地服务不可用提示。
- 对非 JSON 或 JSON 解析失败场景做容错处理。

调整 `frontend/assets/js/form-utils.js`：

- 保留 `showFeedback(message, type)`。
- 新增 `showErrorFeedback(error, fallback)`，由页面统一调用错误格式化逻辑。

### 2.3 页面反馈接线

以下页面脚本已改为使用统一错误反馈：

- `frontend/assets/js/auth-pages.js`
- `frontend/assets/js/admin-management.js`
- `frontend/assets/js/customer-tickets.js`
- `frontend/assets/js/internal-tickets.js`
- `frontend/assets/js/session-ui.js`

同时修复管理页面表单提交顺序：

- 分类新增和客服新增在禁用表单控件前先读取 `FormData`。
- 避免 disabled 控件被 `FormData` 忽略，导致正常提交被误判为空字段。
