# M5-03 开发过程

## 1. 阶段锚定

依据 `docs/04-实施计划/01-MVP实施计划.md`，`M5-03`：确保所有用户文本均以文本形式展示。

`M5-03` 对应验收项：

| 验收项 | 说明 |
| --- | --- |
| `TC-SEC-009` | 在标题、描述或留言输入脚本/HTML 片段并查看页面。内容以文本展示，不执行脚本。 |

对应 API 安全要求：`API-SEC-007` — 页面展示服务端返回的标题、描述和留言时，必须作为文本处理，防止脚本执行。

对应页面实现约束：`文本安全` — 标题、描述、留言等用户输入内容必须按普通文本渲染，不作为 HTML 执行。

## 2. 审查策略

TC-SEC-009 要求验证用户可控制的输入内容在页面展示时不会作为 HTML 执行。审查分为两部分：

1. **后端**：确认用户输入在存储时不做 HTML 转义或过滤（这是正确做法——API 层应保持原始数据，由渲染层负责安全输出）。
2. **前端**：逐一审查所有用户可控内容的 DOM 渲染方式，确认使用安全渲染方法。

### 用户可控内容字段

| 字段 | 来源 | 最大长度 |
| --- | --- | --- |
| `user.username` | 客户注册 / 管理员创建客服 | 50 |
| `user.email` | 客户注册 / 管理员创建客服 | 254 |
| `category.name` | 管理员创建/编辑分类 | 50 |
| `ticket.title` | 客户创建工单 | 100 |
| `ticket.description` | 客户创建工单 | 4000 |
| `message.content` | 客户/客服/管理员留言 | 2000 |

### 安全 vs 危险渲染方法对照

| 方法 | 安全性 | 说明 |
| --- | --- | --- |
| `element.textContent = value` | ✅ 安全 | 浏览器作为纯文本处理，不解析 HTML |
| `document.createElement` + `textContent` | ✅ 安全 | 同上 |
| `innerHTML = escapeHtml(value) + ...` | ✅ 安全（前提是所有动态值都经过 escapeHtml） | 模板字符串需对每个用户值调用 escapeHtml |
| `innerHTML = value` | ❌ 危险 | 浏览器直接解析为 HTML，可执行脚本 |
| `outerHTML` | ❌ 危险 | 同上 |
| `insertAdjacentHTML` | ❌ 危险 | 同上 |
| `document.write` | ❌ 危险 | 同上 |

### 3.5 特殊场景审查

#### `renderEmpty` 函数（admin-management.js:43）

```js
function renderEmpty(tbody, colspan, message) {
  tbody.innerHTML = `<tr><td colspan="${colspan}" class="meta">${message}</td></tr>`;
}
```

`message` 参数仅来自代码内部调用：
- `"暂无分类"`、`"暂无客服账号"`、`"暂无客户账号"`

均为固定中文常量，非用户内容。✅

#### `meta.textContent` 中的插值字符串

customer-tickets.js:250 和 internal-tickets.js:205 中的：
```js
meta.textContent = `${message.sender_name}（${role}） ${formatDateTime(message.sent_at)}`;
```

使用 `textContent`，浏览器将整段字符串作为纯文本处理，即使 `sender_name` 包含 `<script>` 也不会执行。✅

#### `describeAuditChanges` 函数（internal-tickets.js:215-223）

```js
function describeAuditChanges(auditLog) {
  const changes = auditLog.changes || {};
  if (changes.status) {
    return `${statusLabel(changes.status.before) || "无"} -> ${statusLabel(changes.status.after)}`;
  }
  if (changes.assignee_user_id) {
    return `${changes.assignee_user_id.before || "未分配"} -> ${changes.assignee_user_id.after || "未分配"}`;
  }
  return "已记录变更。";
}
```

返回值赋给 `content.textContent`，使用安全渲染。`statusLabel` 为固定枚举映射。`assignee_user_id` 的 before/after 值为 UUID 或 null。✅

#### `statusLabel` 函数（ticket-ui.js）

```js
export function statusLabel(status) {
  return ticketStatusLabels[status] || status;
}
```

`ticketStatusLabels` 为固定枚举映射。当 status 值不在映射中时回退到原始 `status`，但工单状态值来自后端 `TicketStatus` 枚举，不接收用户输入。✅
