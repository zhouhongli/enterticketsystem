# 企业售后工单系统 MVP 测试执行方案

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 适用范围 | 最小可行产品（MVP） |
| 编制日期 | 2026-05-31 |
| 前置条件 | M0-M6 代码开发全部完成，71 个自动化测试全部通过，冒烟测试通过 |

## 1. 测试目标

在不依赖自动化测试框架的前提下，通过**手工操作 + 浏览器 + API 工具**，完成 MVP 五条端到端验收场景的演示，并留存可复用的测试脚本。

## 2. 测试环境

| 项目 | 要求 |
| --- | --- |
| 服务 | `cd backend && .\.venv\Scripts\python.exe -m uvicorn app.main:app --reload` |
| 地址 | http://127.0.0.1:8000 |
| 浏览器 | Chrome 或 Edge（最新稳定版） |
| API 工具 | curl / PowerShell / Postman / 任意 HTTP 客户端 |
| 数据文件 | `backend/data/store.json`（测试前备份，测试后恢复） |
| 环境变量 | `INITIAL_ADMIN_USERNAME`、`INITIAL_ADMIN_EMAIL`、`INITIAL_ADMIN_PASSWORD` |

## 3. 测试数据准备

首次测试时，数据为空。按以下顺序创建初始数据：

| 标识 | 数据 | 创建方式 |
| --- | --- | --- |
| 管理员 | `admin` / `admin12345` | 环境变量配置，首次启动自动创建 |
| 分类 | `产品故障`（启用） | 管理员登录后，在分类管理页新增 |
| 客服 | `agent01` / `agent12345` | 管理员登录后，在客服账号页新增 |
| 客户 | `customer01` / `customer12345` | 客户注册页注册，或管理员在客户账号页查看 |

## 4. 测试场景

### 场景 1：管理员配置 + 客户建单（E2E-01）

**目标**：验证系统初始化、管理配置、客户注册建单主路径。

| 步骤 | 操作 | 预期 | 结果 |
| --- | --- | --- | --- |
| 1 | 打开 http://127.0.0.1:8000/pages/login.html | 显示登录页，含系统名称 | □ |
| 2 | 输入 `admin` / `admin12345`，点击登录 | 跳转到分类管理页 | □ |
| 3 | 在分类管理页，新增分类 `产品故障` | 列表显示该分类，状态为"启用" | □ |
| 4 | 在分类管理页，新增重复分类名称 | 提示"分类名称已经存在" | □ |
| 5 | 在客服账号页，新增客服 `agent01` | 列表显示该客服 | □ |
| 6 | 退出登录，打开注册页 | 显示注册表单 | □ |
| 7 | 注册客户 `customer01` | 提示注册成功，跳转登录页 | □ |
| 8 | 客户登录 | 跳转到客户"我的工单"页 | □ |
| 9 | 点击"新建工单"，选择分类，填写标题和描述 | 工单创建成功，跳转详情 | □ |
| 10 | 查看工单详情 | 状态为"待分配"，负责人为空 | □ |

### 场景 2：分配 + 处理 + 关闭（E2E-02）

**目标**：验证内部处理闭环和状态机。

| 步骤 | 操作 | 预期 | 结果 |
| --- | --- | --- | --- |
| 1 | 退出客户，以 `agent01` 登录 | 跳转到内部工单列表页 | □ |
| 2 | 查看工单列表 | 能看到场景 1 创建的工单 | □ |
| 3 | 退出，以 `admin` 登录 | 跳转到管理页 | □ |
| 4 | 在内部工单列表页，点击工单进入详情 | 显示工单概要、描述、留言区 | □ |
| 5 | 分配工单给 `agent01` | 负责人更新为 agent01，产生操作记录 | □ |
| 6 | 退出，以 `agent01` 登录 | 进入内部工单列表 | □ |
| 7 | 打开工单详情，点击"开始处理" | 状态变为"处理中" | □ |
| 8 | 在工单详情页留言 | 留言显示在留言区 | □ |
| 9 | 点击"标记已解决" | 状态变为"已解决" | □ |
| 10 | 点击"关闭工单" | 弹出确认框，确认后状态变为"已关闭" | □ |
| 11 | 尝试向已关闭工单留言 | 提示"工单已关闭，不能继续留言" | □ |
| 12 | 查看操作记录 | 包含创建、分配、三次状态变更记录 | □ |

### 场景 3：权限隔离（E2E-03）

**目标**：验证非负责人和客户的资源隔离。

| 步骤 | 操作 | 预期 | 结果 |
| --- | --- | --- | --- |
| 1 | 以 `agent01` 登录 | 内部工单列表页 | □ |
| 2 | 注册新客服 `agent02` / `agent12345` | 创建成功 | □ |
| 3 | 以 `agent02` 登录 | 内部工单列表页（非负责人的工单可见） | □ |
| 4 | 打开场景 2 的已关闭工单详情 | 可以看到工单和留言 | □ |
| 5 | 尝试留言（无留言入口或接口） | 不能留言 | □ |
| 6 | 以 `agent02` 身份尝试分配工单（无分配入口） | 无分配操作入口 | □ |
| 7 | 退出，以 `customer01` 登录 | 客户"我的工单"页 | □ |
| 8 | 注册另一客户 `customer02`，登录后尝试通过 API 访问 customer01 的工单 | 返回 404 | □ |

### 场景 4：禁用客户与会话失效（E2E-04）

**目标**：验证客户禁用后自动撤销会话。

| 步骤 | 操作 | 预期 | 结果 |
| --- | --- | --- | --- |
| 1 | 以 `customer01` 登录 | 客户工单页 | □ |
| 2 | 退出，以 `admin` 登录 | 管理页 | □ |
| 3 | 在客户账号页，禁用 `customer01` | 状态变为"禁用" | □ |
| 4 | 以 `customer01` 重新登录 | 提示"账号或密码错误，或账号不可用" | □ |

### 场景 5：重启后数据恢复（E2E-05）

**目标**：验证 JSON 持久化，重启后数据不丢失。

| 步骤 | 操作 | 预期 | 结果 |
| --- | --- | --- | --- |
| 1 | 确认已有数据（分类、客服、客户、工单、留言、操作记录） | 数据完整 | □ |
| 2 | 停止 uvicorn 服务 | 服务停止 | □ |
| 3 | 重新启动服务（同一命令） | 服务正常启动 | □ |
| 4 | 以 `admin` 登录 | 查看分类、客服、客户列表 | □ |
| 5 | 查看分类列表 | 之前创建的分类仍存在 | □ |
| 6 | 查看客服列表 | 之前创建的客服仍存在 | □ |
| 7 | 查看内部工单列表 | 之前创建的工单仍存在 | □ |
| 8 | 打开工单详情 | 留言、操作记录完整 | □ |
